# BOSS 直聘 AI 辅助求职 Chrome 插件设计规范

## 1. 项目概述
本项目是一个无需后端的 Chrome 浏览器插件，旨在利用大语言模型 (LLM) 帮助求职者在 BOSS 直聘平台上实现智能化的寻找岗位、定制化聊天话术生成以及自动化简历投递。

## 2. 核心模式与工作流
插件全面采用 BYOK (Bring Your Own Key) 模式，用户需自行配置大模型提供商的 API Key（如 OpenAI、DeepSeek 等）。简历数据与 LLM 交互全部保留在浏览器本地，确保隐私安全与零服务器成本。

工作流分为两种模式，用户可在插件内自由切换：
*   **模式 A（半自动精投）**：当用户浏览单个岗位页面时，插件通过 Content Script 注入浮动面板，提示当前岗位的分析结果和定制化的大招呼话术。用户人工确认/修改后，点击按钮完成消息发送。
*   **模式 B（全自动海投）**：在搜索列表页，用户圈定搜索条件，启动批量任务。系统将自动抓取列表中的岗位 JD，在后台逐一交由 AI 匹配基础简历，生成应聘话术，并在后台排队完成发起沟通与发送消息操作。

## 3. 架构设计 (Service Worker 主导模式)
采用 Chrome Extension MV3 架构，强调防风控和稳定性：

*   **Service Worker (Background)**:
    *   作为系统“调度大脑”，维护任务队列（如批量海投队列）。
    *   统一封装与 LLM API 的 HTTP 请求，规避跨域问题。
    *   管理本地状态：读取/保存用户基础简历及配置 (Chrome Storage API)。
    *   负责标签页管理：在执行批量投递时，隐式创建新 Tab 进行操作，以防页面刷新中断流程。
*   **Popup / Options 页 (前端 UI)**:
    *   采用轻量级前端框架（如 React + Tailwind CSS）构建。
    *   功能：填写 API Key、管理/编辑多套基础简历。
    *   呈现数据面板：今日海投数量报告、分析消耗 Token 统计等。
*   **Content Script**:
    *   **信息抓取**：安全读取 DOM 节点以提取岗位职责与薪资等信息，上报给 Service Worker。
    *   **界面注入**：采用 Shadow DOM 隔离 CSS，在岗位详情页悬浮展示“AI 投递面板”。
    *   **操作模拟**：模拟真实的人工点击事件 (`dispatchEvent` + 随机延迟) ，将生成的聊天话术填入对话框，降低被平台反作弊系统拦截的几率。

## 4. 关键技术点与挑战应对
*   **Anti-Bot 对策**：所有的自动化点击需要模拟原生的 MouseEvent 进行，点击位置增加像素偏移；批量任务之间必须引入 3-8 秒的随机冷却时间，单日投递强制设置上限。
*   **DOM 解析隔离**：尽量不用 `document.querySelector` 强依赖特定 Class 名（BOSS 平台常常变更），而是采用 `TreeWalker` 或基于文本关键字进行模糊定位。
*   **Context 窗口优化**：JD 与基础简历均可能很长，传递给大模型前需要在插件端进行去重与精简，保证 API 调用不触发限制。

## 5. 开发步骤规划
1.  **基础设施**：搭建 Vite + React + CRXJS (或类似构建工具) 的插件脚手架。
2.  **设置与存储层**：完成 Options 页面及和 Storage的对接（API Key、简历存储）。
3.  **大模型接入层**：实现 Service Worker 中的 fetch API 与 LLM 提供商打通。
4.  **半自动模式 (A)**：抓取单个岗位 JD -> 调用 AI 匹配简历 -> 在页面展示建议话术结果。
5.  **自动化执行**：完成在真实页面写入大招呼文字并触发点击发送交互的逻辑。
6.  **全自动模式 (B)**：开发批量读取搜索列表的队列功能与其循环派发机制。
